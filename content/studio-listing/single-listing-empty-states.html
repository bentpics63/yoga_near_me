Empty States & Partial Data Handling

How each section appears when studio data is missing vs. complete

Data Handling Strategy

Four approaches based on section importance and user value

Hide

Section not shown if empty. No placeholder, no CTA.

Placeholder

Show muted placeholder text indicating data pending.

CTA

Show claim prompt to encourage studio owners to add data.

Collapse

Reduce section size, show partial data available.

Summary: Empty State Strategy by Section

Section Strategy When Empty Reason
Hero Section Collapse Placeholder logo, muted text for missing fields Always needed for page identity
Quick Info Bar Hide Hidden entirely No value if empty, looks broken
Yoga Styles CTA Placeholder with claim prompt Critical for users, encourages claiming
Pricing CTA "Contact for pricing" with phone CTA Users expect pricing info
Teachers Hide Hidden entirely No value without real data
Programs & Services Hide Hidden entirely No value without real data
First Visit Info Hide Hidden entirely Generic info not helpful
Practical Info Collapse Show only populated fields, hide if all empty Partial data still useful
Amenities Placeholder Muted "will be updated" text Keep section visible (existing behavior)
Contact Info Collapse Show only available contact methods Always show what's available
Hours Placeholder "Contact for hours" text Keep section, indicate unknown
Accessibility Hide Hidden entirely Don't assume accessibility features
